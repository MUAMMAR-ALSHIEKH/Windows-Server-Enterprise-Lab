# Day 6 – DNS Configuration & Validation

## Overview
Day 6 focused on configuring and validating *Domain Name System (DNS)* within the Nexora Active Directory environment.  
DNS was implemented as an *Active Directory–integrated service* to ensure reliable name resolution, redundancy, and proper domain functionality.

This step is critical, as Active Directory heavily depends on DNS for authentication, replication, and service discovery.

---

## Environment Details
- *Domain Name:* nexora.local
- *Primary Domain Controller & DNS:* DC01
- *Additional Domain Controller & DNS:* DC02
- *DNS Type:* Active Directory–Integrated DNS
- *Virtualization Platform:* VMware Workstation

---

## Objectives
- Verify Forward Lookup Zone configuration
- Create and validate Reverse Lookup Zone
- Test DNS name resolution
- Validate DNS replication between domain controllers
- Confirm DNS functionality from client machines

---

## DNS Forward Lookup Zone
- Verified the existence of the Forward Lookup Zone:
  - nexora.local
- Confirmed zone type:
  - Active Directory–Integrated
- Verified essential DNS records:
  - Host (A) records for DC01 and DC02
  - SRV records for Active Directory services (_ldap, _kerberos)

---

## DNS Reverse Lookup Zone
- Created an IPv4 Reverse Lookup Zone:
  - Network ID: 192.168.10
- Enabled secure dynamic updates
- Verified PTR records for:
  - DC01
  - DC02
- Ensured proper reverse name resolution

---

## DNS Replication Validation
- Verified DNS zone replication between:
  - DC01
  - DC02
- Confirmed consistency of DNS records across both domain controllers
- Ensured redundancy and high availability of DNS services

---

## Name Resolution Testing
The following commands were used to validate DNS functionality:

```cmd
nslookup nexora.local
nslookup dc01.nexora.local
nslookup dc02.nexora.local

Results

- Successful forward name resolution
- DNS responses served by both domain controllers
- Confirmed AD-integrated DNS replication and redundancy

Client Validation

- Domain-joined client successfully resolved:
. Domain name
. Domain controller hostnames
. Verified DNS dependency for Active Directory services
. No resolution failures or delays observed

Key Observations

-Server: dc01 may appear in nslookup output while the response is served by DC02
. This behavior is normal in multi-DC environments
. Confirms DNS load distribution and failover functionality

- Proper Reverse Lookup Zone configuration resolves Server: Unknown output

Key Takeaways

- DNS is a foundational component of Active Directory
- AD-integrated DNS ensures automatic replication and fault tolerance
- Proper Forward and Reverse Lookup Zone configuration is essential in enterprise environments
- DNS validation is required before deploying services such as DHCP, GPOs, and File Servers
