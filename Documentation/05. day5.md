# Day 5 – Group Policy Management (GPO)

## Overview
In Day 5, Group Policy Objects (GPOs) were implemented and tested to enforce domain-wide and OU-level security restrictions. The goal was to simulate real enterprise policy management, apply restrictions to users and computers, and verify correct policy enforcement.

---

## Objectives
- Create and manage Group Policy Objects
- Apply GPOs at the Organizational Unit (OU) level
- Enforce user-based restrictions
- Verify policy application on client machines
- Document and validate results using built-in Windows tools

---

## Tasks Completed

### 1. Group Policy Object Creation
- Created a custom GPO named:
  - *NEXORA-User-Restrictions*
- GPO was linked directly to the appropriate Organizational Unit (OU)
- Avoided modifying Default Domain Policy.

---

### 2. Policy Configuration
The following user-based restrictions were configured through Group Policy Management:

- Control Panel access restrictions
- Administrative feature limitations
- User environment lockdown settings
- Security-focused user configuration policies

All policies were set to *Enabled* and scoped correctly to target users.

---

### 3. OU and Computer Placement
- Verified that target computers were placed inside the correct OU:
  - *Workstations*
- Ensured users were located in the correct department OU (e.g., IT)
- Confirmed no devices remained incorrectly under the default Computers container

---

### 4. Policy Application and Testing
On the client workstation:

- Forced policy update:
  ```powershell
  gpupdate /force
 
- Verified applied policies using:
  ```powershell
  gpresult /r
